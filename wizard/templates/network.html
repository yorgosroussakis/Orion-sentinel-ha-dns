Configure Your Network

Single-Node: Best for most home users. All services run on one Pi with automatic container failover.
Two-Pi HA: Maximum availability. Survives complete Pi failure. Requires 2 Raspberry Pis.

The static IP address of THIS Raspberry Pi on your LAN.
Two-Pi HA: Pi1=192.168.8.11, Pi2=192.168.8.12 (example)

Usually 'eth0' for wired or 'wlan0' for wireless

üìã Two-Pi HA Setup: You'll configure this wizard on BOTH Raspberry Pis separately.
  • On Pi1: Set Node Role to Primary, enter Pi1's IP
  • On Pi2: Set Node Role to Secondary, enter Pi2's IP
  • Use the same VIP and passwords on both Pis!

The shared "floating" IP that moves between Pis during failover.
Requirements:

  • Must be in the same subnet as your Pis
  • Must NOT be in your router's DHCP range
  • Must NOT be .0 (network) or .255 (broadcast)
  • Must be SAME on both Pi1 and Pi2
All LAN clients will use ONLY this IP for DNS.

The IP address of the OTHER Raspberry Pi.
On Pi1, enter Pi2's IP. On Pi2, enter Pi1's IP.

Primary: This Pi will normally own the VIP (higher priority = 200)
Secondary: Takes over VIP when Primary fails (lower priority = 150)

Password for VRRP communication between nodes (minimum 8 characters).
CRITICAL: Must be the SAME on both Pi1 and Pi2!

Minimum 8 characters. Used to access the Pi-hole web interface.
